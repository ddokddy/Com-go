HTML Tag & 문법 List

1. 글자굵게

strong , /strong

2. 밑줄치기

u, /u

3. 단락지정

p, /p

4. 제목설정

h1~h6 - /h1~/h6

5. 줄 바꿈

br

6. 이미지넣기

img src="파일위치" width=%또는px

src와 width는 순서 상관없음(width가 src 전에 와도 됨)

7. 리스트

li, /li (list의 앞 2자)

8. 리스트의 부모태그

li 태그는 무조건 li를 그룹화 하는 태그인 ul 태그가 필요함.
li의부모 -> ul ul,/ul (리스트의 약자가 l)
unordered list 의 약자
반대로 ul 태그는 반드시 li가 필요함

9. 넘버링

ol,/ol (리스트의 약자가 l) = li 를 그룹화하여 넘버를 매겨줌
ordered list 의 약자.

li는 2대가 따라다님. li-ul-ol

10. 웹문서타이틀

title, /title - 웹문서 탭에 표시되는 제목
검색엔진은 title 의 태그에 할당되는 검색어를 책표지와 같은 정보로 이해.
title태그를 이용하지 않는 것은 굉장히 큰 손해이다.

11. 타 브라우저 또는 에디터에서 저장한 한글이 제대로 표시 안될때

meta charset="utf-8" - 이 html 문서를 utf-8로 읽어라. = 한글 표시

12. html의 구조

!doctype html관용적으로 사용
html-head-head-body-body-html

태그의 제왕

이 태그 덕분에 웹페이지를 발견할 수 있고,
이 태그가 있기에 검색엔진이 존재한다.
도시의 길과, 몸의 혈관같은존재이다
일상에서 하루에 100번도 넘게 사용하는 태그
html 의 약자, hyper text가 이 태그를 의미한다.

이 태그는 a 이다.

정보의 바다에 정박한다는 의미의 anchor의 첫글자를 땀.
a는 링크의 기능을 한다.
